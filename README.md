# Smash Bracket — Render Deployment Guide

## Project structure

```
smash-bracket/
├── render.yaml              ← Render blueprint (deploys everything at once)
├── backend/
│   ├── main.py              ← FastAPI: auth, brackets, leaderboard
│   ├── database.py          ← SQLAlchemy models (User, Bracket, RoundRobinResult)
│   └── requirements.txt
└── frontend/
    ├── main.py              ← Streamlit app (updated with login + API calls)
    ├── requirements.txt
    └── images/              ← (optional) your character icon PNGs go here
```

---

## Local development

### 1. Start the backend
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
# API runs at http://localhost:8000
# Swagger docs at http://localhost:8000/docs
```

### 2. Start the frontend
```bash
cd frontend
pip install -r requirements.txt
export BACKEND_URL=http://localhost:8000
streamlit run main.py
```

---

## Deploy to Render (one-click blueprint)

1. Push this repo to GitHub (or GitLab).
2. Go to **render.com → New → Blueprint**.
3. Connect your repo — Render will detect `render.yaml` automatically.
4. Click **Apply** — Render will create:
   - `smash-bracket-db` (PostgreSQL, free tier)
   - `smash-bracket-api` (FastAPI web service)
   - `smash-bracket-ui` (Streamlit web service)
5. Done. Render wires `DATABASE_URL` and `BACKEND_URL` together automatically.

## Environment variables (already handled by render.yaml)

| Variable | Where | Notes |
|---|---|---|
| `SECRET_KEY` | backend | Auto-generated by Render |
| `DATABASE_URL` | backend | Auto-filled from the Postgres DB |
| `BACKEND_URL` | frontend | Auto-filled from the API service's internal URL |

## Character icons

To keep character icons working on Render:

1. Put your PNGs in `frontend/images/` (same filenames as before, e.g. `Mario.png`).
2. Add this to your Streamlit config (`frontend/.streamlit/config.toml`):
   ```toml
   [server]
   enableStaticServing = true
   ```
3. Move images to `frontend/static/images/` — Streamlit serves `/app/static/` automatically.
4. The `get_character_icon_url()` function in `main.py` already returns the correct URL path.

## What changed from the Streamlit-only version

| Before | After |
|---|---|
| State in `st.session_state` (lost on refresh) | Saved to PostgreSQL via API |
| No accounts | JWT login/signup in the sidebar |
| No history | "My Brackets" and "My RR Sessions" pages |
| No global stats | "Global Leaderboard" page |
| `file://` icon paths (broken on server) | URL-based static file serving |
| All logic in one file | backend/ handles data, frontend/ handles UI |
